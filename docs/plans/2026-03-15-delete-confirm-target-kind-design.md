# 删除确认目标类型设计

**背景**

会话执行 `file_delete` 时，高风险确认弹窗当前统一显示“删除文件”。当实际目标是空文件夹或递归删除目录时，用户会被错误文案误导，以为系统删除了错误对象。

**问题定位**

- `file_delete` 工具本身支持删除文件或目录。
- 后端 [`critical_action_summary`](../../apps/runtime/src-tauri/src/agent/executor.rs) 针对 `file_delete` 写死返回“删除文件”。
- 前端只是展示后端发来的标题和摘要，并不是根因。

**目标**

- 文件目标显示“删除文件”。
- 空目录目标显示“删除文件夹”。
- `recursive=true` 的目录目标显示“递归删除文件夹”。
- 无法确定目标类型时回退到“删除目标”，避免误导。

**方案**

1. 在后端生成确认事件时，根据 `path` 和当前 `work_dir` 解析目标实际路径。
2. 用文件系统元信息判断目标是文件、目录或未知。
3. 按目标类型生成更准确的标题、摘要和影响说明。
4. 用单元测试覆盖文件、目录、递归目录和未知目标四种情况。

**非目标**

- 不改前端弹窗布局。
- 不新增前端目标类型推断逻辑。
- 不改 `file_delete` 工具的执行行为。
