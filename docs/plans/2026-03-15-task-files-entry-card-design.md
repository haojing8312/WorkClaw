# 任务文件入口卡片视觉优化设计

**日期：** 2026-03-15

## 背景

对话完成后，聊天转录区会在对应消息下方展示一个入口按钮：`查看此任务中的所有文件`。当前实现虽然功能正确，但视觉上更像一个被放大的默认按钮：

- 字号过大，像页面标题，不像消息流中的次级入口
- 留白过多，导致卡片显得空且笨重
- 图标、标题、边框、阴影都缺少层次
- 缺乏“任务已交付”的完成态反馈，只表达了“可点击”

这让它在当前聊天界面中显得突兀，也没有很好地承接“任务已完成，点击查看产物”的心理预期。

## 目标

将该入口从“放大按钮”优化为“完成态交付卡片”：

- 保留现有文案和点击行为
- 在视觉上更贴近聊天流中的完成结果卡片
- 增强点击引导，但不喧宾夺主
- 与现有蓝白色系和圆角卡片语言保持一致

## 方案比较

### 方案 A：完成态卡片 `推荐`

采用浅蓝到白色的轻渐变卡片，左侧放图标徽章，中间使用主副标题信息结构，右侧增加方向箭头。

优点：

- 明确传达“任务已完成，可查看交付物”
- 信息层次更完整，观感更精致
- 与当前产品的蓝白色视觉语言自然融合

缺点：

- 比纯文本入口更强调视觉存在感

### 方案 B：轻量列表入口

保留单行信息结构，只收紧边框、字号和 hover。

优点：

- 改动最小
- 视觉最克制

缺点：

- 改善有限
- 在聊天区留白较大时仍可能显得单薄

### 方案 C：主操作按钮

强化主按钮感，使用更重的品牌色与交互反馈。

优点：

- 点击意图最强

缺点：

- 太像 CTA，容易抢走正文和输入框的注意力

## 推荐方案

采用方案 A，把当前入口升级为一张轻量的完成态交付卡片。

## 最终设计

### 1. 整体结构

入口仍然是一个 `button`，但内部结构改为三段：

- 左侧：浅蓝徽章中的文件夹图标
- 中部：主标题 + 副标题
- 右侧：轻量箭头图标

### 2. 视觉语言

- 外层使用更精致的圆角和更轻的阴影，避免现在的大白块感
- 背景使用极浅的蓝白渐变，增加完成态和质感
- 边框改为偏蓝灰色，hover 时稍微提亮
- 卡片高度收紧，减少冗余留白

### 3. 文案层次

保留主标题：

- `查看此任务中的所有文件`

新增副标题：

- `任务已完成，点击查看本次产出文件`

如果 `deliverables.length > 0`，补充数量提示，例如：

- `共 3 个文件`

### 4. 交互反馈

- hover：边框变亮、阴影轻微增强、整体略微上浮
- active：轻微回落
- focus-visible：补清晰的蓝色 focus ring
- 保留完整按钮语义，不改变现有点击行为

### 5. 响应式与可访问性

- 卡片最小点击高度保持舒适点击区域
- 在窄屏下副标题允许换行，避免主标题拥挤
- 图标仅作装饰，按钮可访问名称继续由可见文案提供

## 影响范围

- 主要修改：`apps/runtime/src/components/chat-journey/TaskJourneySummary.tsx`
- 测试更新：`apps/runtime/src/components/__tests__/ChatView.side-panel-redesign.test.tsx`

## 非目标

- 不调整点击后的侧边栏行为
- 不新增新的交付摘要逻辑
- 不改动“何时显示此卡片”的完成态判定
