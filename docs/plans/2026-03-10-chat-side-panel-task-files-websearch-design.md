# Chat Side Panel Task Files WebSearch Design

**背景**

当前聊天右侧面板同时承载了附件列表、工具调用历史、自动路由、IM 协作时间线等多种能力，数据来源混杂在本地输入状态、消息持久化记录和运行时事件之间。结果是用户无法快速理解“这一轮任务正在做什么”“AI 生成了哪些文件”“这轮搜索到了什么结果”。

本次重构目标是删除旧侧栏能力，重新聚焦为三类高价值信息：

- 当前任务
- 文件
- Web 搜索

设计原则：

- 当前任务是会话级过程视图
- 文件是工作空间级资产视图
- Web 搜索是会话级研究视图
- 会话主区域保留消息内工具回放，不再承担侧栏信息总览

## 用户目标

用户在会话执行期间需要快速回答三个问题：

1. AI 当前在做什么，还剩什么没做
2. AI 在当前工作空间里产出了什么文件，能否立即预览
3. AI 搜索了什么，结果是否可信，能否安全打开原始链接

## 信息架构

右侧面板替换为 3 个 tab：

- `当前任务`
- `文件`
- `Web 搜索`

移除以下旧能力：

- `附件与工具`
- `自动路由`
- 侧栏内附件列表
- 侧栏内工具调用历史
- 侧栏内 IM 协作时间线
- 侧栏内路由决策
- 侧栏内调用链

会话气泡区保留 `ToolIsland`，用于消息内部的时序回放。

## Tab 1：当前任务

### 目标

将 `todo_write` 变成面向用户可读的任务执行面板，而不是工具日志。

### 展示结构

1. 顶部摘要卡
   - 总任务数
   - 已完成数
   - 进行中数
   - 当前执行项标题
2. 任务列表
   - 每项显示状态、标题
   - `in_progress` 高亮
   - `completed` 勾选
   - `pending` 弱化
3. 本轮动作摘要
   - 本轮生成文件数
   - 本轮 Web 搜索次数
   - 最近一次文件产出名
   - 最近一次搜索 query

### 数据来源

主来源：

- 会话消息中的 `todo_write` 工具调用

派生来源：

- 会话消息中的 `write_file`
- 会话消息中的 `edit`
- 会话消息中的 `web_search`

### 降级策略

若本轮会话没有出现 `todo_write`：

- 顶部显示 `未创建任务清单`
- 不推断虚假任务
- 仅展示“本轮动作摘要”

这保证信息可信，不用猜测 Agent 意图。

## Tab 2：文件

### 目标

提供工作空间全量文件浏览与可用预览，而非只显示输入附件。

### 布局

- 左栏：文件树 / 文件列表
- 右栏：预览区
- 顶部工具区：
  - 搜索框
  - 在系统文件夹打开
  - 筛选：全部 / 本会话产出 / 文档 / 网页 / Markdown / 代码

### 范围定义

- 展示当前工作空间全量文件
- 本会话创建或修改的文件加 `本轮生成` 或 `本轮更新` 标识

### 文件预览能力

纯文本与源码类：

- `.md`
- `.txt`
- `.json`
- `.js`
- `.ts`
- `.tsx`
- `.css`
- `.html`

能力：

- 源码预览
- 行滚动
- 复制

渲染预览：

- `.md`：Markdown 渲染
- `.html`：页面预览 + 源码预览双模式

办公文档：

- `.docx`：显示文件元信息，支持在系统默认应用打开
- 后续若已有抽取能力，可扩展文本预览，但不作为首版阻塞项

其他二进制：

- 显示文件大小、修改时间、扩展名
- 提供打开操作

### HTML 预览体验

HTML 是高价值场景，必须单独设计：

- 预览区使用模式切换：`页面` / `源码`
- `页面` 模式用隔离 iframe 渲染
- `源码` 模式保持等宽字体与高密度阅读
- 若文件引用相对资源，优先基于工作空间路径解析
- 若渲染失败，展示明确错误信息并回退源码模式

## Tab 3：Web 搜索

### 目标

把 `web_search` 从“工具调用记录”升级为“可回看的研究面板”。

### 布局

- 左栏：搜索记录列表
- 右栏：搜索详情

搜索记录显示：

- query
- 时间
- 状态
- 结果数

搜索详情显示：

- query
- 原始输出摘要
- 结果列表

每条结果展示：

- 标题
- 来源域名
- snippet
- URL

### 交互

- 点击搜索记录，右侧切换对应详情
- 点击搜索结果，弹出确认框
- 用户确认后，调用后端打开系统浏览器

### 边界

- 仅显示当前会话内发生过的 Web 搜索
- 相同 query 多次调用也分别保留
- 不与工作空间全局历史混合

## 数据模型

建议从 `ChatView` 中抽离新的侧栏聚合层，统一生成：

- `TaskPanelViewModel`
- `WorkspaceFilePanelViewModel`
- `WebSearchPanelViewModel`

其中：

- `TaskPanelViewModel` 基于会话 `streamItems/toolCalls`
- `WorkspaceFilePanelViewModel` 基于当前 `workspace` 的文件扫描结果 + 本会话写文件记录
- `WebSearchPanelViewModel` 基于会话内 `web_search` 调用输出解析

建议新增独立组件：

- `TaskPanel`
- `WorkspaceFilesPanel`
- `WebSearchPanel`
- `FilePreviewPane`
- `OpenExternalUrlDialog`

## 前后端职责划分

前端：

- 聚合会话工具调用为任务和搜索视图
- 驱动 tab、筛选、预览、选择态

后端：

- 提供工作空间文件树查询
- 提供文件内容读取与元信息读取
- 提供 HTML 预览资源解析所需最小能力
- 提供安全打开外链命令

建议新增 Tauri 命令：

- `list_workspace_files`
- `read_workspace_file_preview`
- 复用已有 `open_external_url`

## 测试策略

前端测试应覆盖：

- 有 `todo_write` 时正确显示任务统计
- 无 `todo_write` 时显示空任务态
- 文件 tab 正确渲染工作空间文件并标记本会话产出
- HTML 文件支持页面与源码切换
- Web 搜索 tab 正确展示多次搜索记录与详情
- 点击搜索结果弹确认，再触发外链打开
- 旧 tab 与旧侧栏内容不再出现

后端测试应覆盖：

- 工作空间文件枚举排序与过滤
- 文件预览读取限制
- HTML / Markdown / 文本文件预览返回结构
- 非文本文件回退为元信息模式

## 非目标

本次不做：

- `.docx` 内嵌富文本渲染器
- 全工具统一日志中心
- 自动路由信息迁移到新侧栏
- 会话外 Web 搜索历史聚合

## 迁移策略

首版采用替换策略：

- 新侧栏直接替换旧侧栏内容
- 旧自动路由摘要按钮一并移除
- 保留主消息区的 `ToolIsland`

这样可以最大限度减少双系统并存带来的认知负担。
