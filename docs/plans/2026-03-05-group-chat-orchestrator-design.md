# 多员工“拉群协作”编排器设计（10人、协调员轮次）

## 1. 目标

在桌面端支持用户创建“员工群组”（最多 10 人），指定 1 名协调员。用户发送单条指令后，系统必须完成：

1. 协调员生成可执行计划
2. 协调员按轮次调度成员执行
3. 协调员输出最终汇报（含完成项/未完成项/风险）

## 2. 关键约束

- 群组成员上限：10
- 发言机制：仅协调员调度，成员按轮次响应
- 任务完成标准：必须包含“计划 + 自动执行 + 汇报”三阶段
- 不引入成员自由互聊

## 3. 架构决策

1. 新增 Group Orchestrator 作为中心状态机，不让提示词“自由发挥”
2. 复用现有 `task` 工具执行子任务，统一委托入口
3. 增加群组会话模型，和现有单员工会话并存

## 4. 数据模型

- `employee_groups`
  - `id`, `name`, `coordinator_employee_id`, `member_employee_ids_json`, `created_at`
- `group_runs`
  - `id`, `group_id`, `session_id`, `user_goal`, `state`, `current_round`, `created_at`, `updated_at`
- `group_run_steps`
  - `id`, `run_id`, `round_no`, `assignee_employee_id`, `step_type(plan|execute|review)`, `input`, `output`, `status`, `started_at`, `finished_at`

## 5. 状态机

- `planning`：协调员生成结构化计划（任务、负责人、依赖、验收）
- `executing`：按轮次调度成员执行；支持并发窗口（默认 3）
- `synthesizing`：协调员聚合结果并补漏
- `done|failed`

失败策略：

- 单成员超时：重试 1 次后降级为“未完成项”
- 单成员失败：不中断整体 run，由协调员给替代方案
- 连续失败过多：run 标记 `failed`，输出失败报告

## 6. UI 设计

1. 员工页新增“拉群协作”
   - 多选成员（最多 10）
   - 选择协调员
   - 保存群模板
2. 聊天页新增“协作看板”
   - 阶段：计划/执行/汇总
   - 轮次：第 N 轮
   - 成员状态：进行中/完成/失败/超时
3. 消息区显示
   - 协调员指令消息
   - 成员轮次回复消息
   - 最终汇报卡片

## 7. 与现有能力映射

- 已有：`TaskTool`（子 Agent 执行 + 角色标注 + 流式）
- 已有：协作提示注入（可委托员工列表）
- 已有：主/子员工前端可视化
- 待补：群组实体、轮次状态机、协调员编排协议、拉群 UI

## 8. 验收标准

1. 可创建 10 人以内群组并指定协调员
2. 用户一句话触发后，必定经历计划/执行/汇报三阶段
3. 成员按轮次响应，协调员负责调度
4. 失败成员不会中断整体流程，最终报告可见未完成项与补救建议
