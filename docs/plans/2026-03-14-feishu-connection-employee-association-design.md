# WorkClaw 飞书连接与员工关联设计

## 背景

WorkClaw 当前已经具备飞书消息接入、路由分发和员工协作能力，但在产品表达上，"飞书接入" 与 "员工分配" 容易被用户理解为同一件事。飞书官方 OpenClaw 插件发布后，这种混淆会进一步放大：用户会不清楚到底是先接飞书，还是先配员工，也不清楚两套配置是否重复。

为了降低理解成本，同时保留 WorkClaw 的差异化能力，本方案将飞书相关能力拆成两个连续步骤：

- 飞书连接：解决 "入口是否接通"
- 员工关联：解决 "消息由谁处理"

## 目标

- 把飞书连接从员工配置流程中拆出来，作为独立配置域
- 让用户先完成飞书连接，再将飞书入口关联给某个员工
- 保留 WorkClaw 的核心差异化：默认接待员工、按群聊或会话分配、员工协作编排
- 保持默认路径简单，不引入复杂模式心智

## 非目标

- 不在本期突出多机器人专属模式
- 不重写飞书底层接入逻辑
- 不引入组织级审批、审计、权限分层

## 产品原则

- 连接与分配分离
- 默认路径优先简单
- 高级配置按需展开
- 飞书只是入口，员工协作才是 WorkClaw 的核心价值

## 信息架构

- 设置中心
  - 渠道连接
    - 飞书连接
- 智能体员工
  - 员工详情
    - 飞书接待

## 核心概念

### 飞书连接

一个可用的飞书接入配置，包含插件状态、机器人绑定、授权状态和基础开关。

### 默认接待员工

某个飞书连接的兜底处理人。当消息未命中任何指定范围规则时，交由该员工处理。

### 指定处理范围

某员工只处理某些群聊、会话或线程的飞书消息。

### 员工关联

员工与某个飞书连接之间的处理关系，包含是否接待、接待范围和优先级。

## 核心规则

- 一个飞书连接可关联多个员工
- 一个飞书连接只能有一个默认接待员工
- 指定范围规则优先于默认接待
- 多条指定范围规则冲突时按优先级命中
- 没有命中规则时回退给默认接待员工
- 若既无命中规则也无默认接待员工，则进入未分配状态并提示用户补齐配置

## 页面设计

### 页面一：设置中心 > 渠道连接 > 飞书连接

#### 页面目标

- 让用户完成飞书接入，不承载员工分工逻辑
- 清晰展示当前连接状态与下一步操作

#### 模块顺序

1. 总状态卡
2. 快速操作区
3. 授权引导区
4. 基础开关区
5. 关联概览区

#### 展示字段

- 连接名称
- 机器人名称
- 状态：未连接 / 待授权 / 已连接 / 异常
- 插件版本
- 最近检测时间
- 默认接待员工
- 已关联员工数

#### 核心操作

- 安装插件
- 关联已有机器人
- 新建机器人
- 完成授权
- 验证连接
- 查看日志

#### 基础开关

- 流式输出
- 仅 @ 回复
- 话题独立上下文

#### 关键文案

- 飞书连接成功后，你就可以在飞书中给 WorkClaw 分配任务
- 默认接待员工负责处理未命中规则的飞书消息
- 已关联员工数仅用于概览，不在此页直接编辑

### 页面二：智能体员工详情 > 飞书接待

#### 页面目标

- 决定该员工是否处理某个飞书连接的入口消息
- 支持默认接待与指定范围两类关联方式

#### 模块顺序

1. 启用飞书接待
2. 选择飞书连接
3. 配置接待方式
4. 指定处理范围
5. 冲突提示

#### 核心配置

- 启用飞书接待
- 关联飞书连接
- 接待方式
  - 设为默认接待员工
  - 仅处理指定群聊或会话

#### 指定处理范围字段

- 群聊或会话 ID
- 备注名称
- 优先级

#### 关键文案

- 开启后，该员工可以处理来自飞书连接的消息
- 默认接待员工负责兜底处理未命中规则的飞书消息
- 指定群聊或会话的规则优先于默认接待生效

## 默认用户路径

1. 用户进入设置中心的飞书连接页
2. 安装插件并完成机器人绑定
3. 根据引导完成授权与验证
4. 系统提示用户选择一个默认接待员工
5. 用户前往员工详情页设置默认接待员工
6. 配置完成后，飞书入口具备最小可用闭环

## 进阶用户路径

1. 用户已具备一个可用飞书连接
2. 打开某员工详情页，启用飞书接待
3. 选择同一个飞书连接
4. 仅配置该员工处理某些群聊或会话
5. 保存后，该员工优先处理命中的范围，其余消息仍回退给默认接待员工

## 空状态与异常状态

### 飞书连接页空状态

- 还没有飞书连接
- 完成飞书连接后，你就可以在飞书中给 WorkClaw 分配任务

### 员工页空状态

- 该员工尚未关联飞书入口
- 关联后，可直接处理来自飞书的消息和任务

### 异常状态

- 飞书连接出现异常，请重新验证或检查插件状态
- 当前授权已失效，请重新完成飞书授权
- 当前处理范围与其他员工重叠，请调整范围或优先级后再保存
- 该消息未命中任何员工规则，且没有默认接待员工

## 价值总结

本方案的核心不是新增一套飞书能力，而是澄清飞书能力在 WorkClaw 中的产品定位：

- 飞书连接负责把入口接进来
- 员工关联负责把入口分给合适的人

这样既能消除重复配置心智，也能放大 WorkClaw 作为员工协作编排平台的差异化价值。
