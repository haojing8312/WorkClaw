# 飞书配置单入口（智能体员工）设计

## 背景

当前产品在两个位置暴露飞书相关配置：

- `智能体员工`（员工级：`open_id`、`App ID`、`App Secret`）
- `设置 -> 飞书协作`（网关/连接/路由/线程绑定，同时也包含员工配置）

这导致普通用户难以判断“去哪里配置才生效”，形成重复入口和高认知负担。

## 目标

- 飞书配置只保留一个用户入口：`智能体员工`。
- 普通用户不再看到网关、线程、路由等高级概念与操作。
- 保持已有员工级飞书凭据配置能力不变（`App ID`、`App Secret`、`open_id`）。

## 方案

采用方案 A（收敛入口）：

1. 移除设置页顶部 Tab 中的 `飞书协作` 入口。
2. 移除设置页 `activeTab === "feishu"` 的整个渲染区块（网关配置、连接状态、员工管理副本、路由向导、线程绑定）。
3. 保留侧边栏一级导航 `智能体员工`，作为唯一飞书凭据配置入口。
4. 更新 README 中飞书接入说明，改为“在智能体员工中为员工绑定飞书凭据”。
5. 调整前端测试，验证设置页不再展示飞书协作入口。

## 非目标

- 不改动后端飞书网关/路由能力实现。
- 不改动智能体员工页字段结构与保存逻辑。
- 不在本次新增“开发者模式”。

## 风险与缓解

- 风险：旧测试用例依赖 `飞书协作` Tab，会失败。
  - 缓解：同步更新相关测试为“入口已移除”的新预期。
- 风险：文档与行为不一致。
  - 缓解：同次提交更新中英文 README 的飞书配置步骤。

## 验收标准

- 设置页不再出现 `飞书协作` Tab。
- 用户仍可在 `智能体员工` 页面配置 `open_id/App ID/App Secret`。
- 相关前端测试通过。
- README 飞书配置说明不再指向“设置 -> 飞书协作”。
