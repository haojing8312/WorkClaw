## 为什么要看这一页
WorkClaw 涉及模型 Key、文件目录、技能安装和执行行为。  
只要遵守下面规则，日常风险可以显著降低。

## 五条必遵守规则
1. 只安装可信来源的技能包和技能目录。
2. 只给应用配置工作相关目录，不给全盘目录。
3. 删除、覆盖、清空记忆前先二次确认对象。
4. API Key 不外发，不截图明文，不在群里粘贴。
5. 不确定时先停手，提交问题再继续。

## 高风险动作前的自检
在执行以下动作前，先自检一次：
- 移除技能
- 删除员工
- 清空长期记忆
- 切换高风险自动化模式

当前个人版默认使用“标准模式”：
- 大部分操作会直接执行
- 只有删除、永久覆盖、工作区外写入、外部提交等极高危动作才会再次确认

如果你主动切换到“全自动模式”：
- 所有操作都会自动执行
- 适合可信任务与熟悉环境
- 切换前请务必确认当前任务和目录范围正确

自检问题：
1. 我是否确认目标对象是正确的？
2. 我是否确认这一步可逆？
3. 我是否保留了必要备份？

## 账号与密钥建议
1. 不同环境使用不同 API Key。
2. 定期轮换 Key。
3. 离职或角色变更后及时失效旧 Key。
