# WorkClaw 发布前测试用例表

## 1. 文档用途

本文档用于把专业测试方案进一步落成可执行的发布前测试用例，适用于：

- 发布前测试执行
- 修复后的回归验证
- 多人协作测试时统一记录口径

建议配合下面文档一起使用：

- [11-professional-test-plan.md](./11-professional-test-plan.md)
- [10-non-technical-test-record-templates.md](./10-non-technical-test-record-templates.md)

## 2. 执行说明

每条用例建议记录以下字段：

- 执行人
- 执行日期
- 测试版本
- 环境
- 实际结果
- 是否通过
- 缺陷编号
- 备注

通过状态建议统一为：

- `Pass`
- `Fail`
- `Blocked`
- `Not Run`

## 3. 环境准备

执行前建议准备：

- 干净安装环境
- 已配置模型和搜索的环境
- 一个可读写的默认工作目录
- 一组本地测试文件
- 至少一个可安装技能
- 至少一个员工样本
- 至少一个团队样本

## 8. 建议执行顺序

建议按以下顺序执行：

1. 执行全部 P0 冒烟用例
2. 若 P0 全通过，再执行 P1
3. 若主流程稳定，再执行 P2
4. 修复问题后优先回归相关 P0/P1 用例

## 9. 使用建议

如果你只有有限时间，优先顺序如下：

1. TC-P0-001 到 TC-P0-009
2. TC-P1-004 到 TC-P1-012
3. TC-P1-001 到 TC-P1-003
4. 全部 P2

如果你要用于正式发布门槛，建议至少做到：

- 全部 P0 执行
- 全部 P1 执行
- P0 和 P1 中无阻断与严重问题
