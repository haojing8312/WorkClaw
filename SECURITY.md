# Security Policy

## Supported Versions

WorkClaw is under active development. Security fixes are prioritized for:

- `main` (latest source)
- latest released version (when available)

Older snapshots or unmaintained forks may not receive security updates.

## Reporting a Vulnerability

Please do **not** report security vulnerabilities through public GitHub issues.

Preferred channel:

- Open a private GitHub security advisory:
  `https://github.com/haojing8312/WorkClaw/security/advisories/new`

If private advisory is unavailable, contact the maintainer through GitHub profile and avoid sharing exploit details publicly.

## What to Include

- Affected component and version/commit
- Impact assessment
- Steps to reproduce (proof of concept if possible)
- Suggested fix or mitigation (optional)

## Response Process

- We will acknowledge receipt as soon as possible.
- We will validate impact and triage severity.
- We will coordinate a fix and disclosure timeline.

## Safe Harbor

If you act in good faith and avoid privacy/service abuse, we will treat your report as responsible disclosure.
